# Technical Architecture Documentation

Use this reference for architecture overviews, system context, C4 container,
component, deployment documentation, and architecture-facing explanatory pages.

## Required Sources

- Local code, package boundaries, deployment manifests, infrastructure code,
  configs, diagrams, ADRs, and existing README files.
- C4 model guidance for diagram level and audience fit:
  https://c4model.com/
- Writerside structure and diagram embedding rules from
  `writerside-technical-documentation.md`.

## Default Paths

- `docs/architecture/architecture-overview.md`
- `docs/architecture/system-context.md`
- `docs/architecture/c4/container-model.md`
- `docs/architecture/c4/component-model.md`
- `docs/architecture/c4/deployment-model.md`
- diagrams in `docs/diagrams/` as `.drawio` source plus `.svg` export

## Required Content

- System purpose and boundaries.
- Primary users, external systems, data stores, and integration points.
- Major runtime components and responsibilities.
- Critical data flows and trust boundaries.
- Deployment environments where topology changes behavior.
- Constraints, assumptions, and known unknowns.
- Links to ADRs for locked decisions.
- Links to current diagrams generated by `technical-diagrams`.

## Workflow

1. Identify the architecture level: context, container, component, deployment,
   or overview.
2. Read local source first. Do not infer architecture from file names alone
   when code, configs, or deployment manifests are available.
3. Route diagram creation or refresh to `technical-diagrams` when structure,
   data flow, permissions, or deployment topology must be visual.
4. Keep prose explanatory. Put step-by-step deployment or operations details in
   operations docs.
5. Link to relevant ADRs instead of re-arguing decisions.
6. Mark sensitive infrastructure, threat, or tenant information as Restricted.

## Validation

- Every named component has evidence in code, config, deployment, or a stated
  assumption.
- Diagram labels match prose and source names.
- External systems and trust boundaries are explicit.
- Outdated diagrams are not embedded as current.
- The topic appears in the correct Writerside tree.
